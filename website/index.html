Quick start

The actual pairing flow is browser first, command second.

01

Start WinBridge AI on Mac

Run the local server on your Mac. The terminal prints LAN URLs that Windows can open.

npm start

# Example printed URL:
http://192.168.0.110:47832/
02

Open the Mac URL on Windows

Use a Windows browser to open the printed URL. In LAN Agent, copy the generated command and run it in PowerShell.

LAN Agent Ready to connect this Windows session
Copy Agent Command
03

Operate Windows from Mac

Keep the Windows PowerShell agent open. Back on Mac, use the web console, CLI, or AI tools to run real Windows work.

npm run status
npm run run -- "hostname; whoami"
npm run screenshot

Token entry is intentionally off by default for trusted LAN pairing. Enable token mode only when you need a stricter local session.

How it works

One bridge, two machines, a full Windows operation surface.

01

Start on Mac

Run the local server. It prints LAN pairing URLs and keeps all runtime state on your Mac.

02

Pair Windows

Open the URL on Windows and run the generated PowerShell agent command. The agent polls the Mac.

03

Let AI operate

Use the web console, CLI, and future MCP tools to operate Windows through commands, files, processes, logs, screenshots, and task templates.

Not just builds. Operate the machine.

Builds and installers are just examples. The product direction is a general Windows control surface for whatever the connected user permissions and automation tools allow.

  • Run commands and scripts
  • Inspect files, env, logs, and processes
  • Capture screenshots for visual feedback
  • Move files and operate artifacts
  • Launch apps, installers, tests, and tools
  • Expose the surface to AI agents

Security model

Local-first by default, token mode when you need it.

WinBridge AI is designed for trusted local networks first. The default pairing flow avoids manual token entry on Windows, while optional token mode can protect the console, API, and agent callbacks when you need a stricter LAN session. 

WINBRIDGE_PORT=47832
WINBRIDGE_STATE_DIR=.state
WINBRIDGE_AUTH_REQUIRED=1
