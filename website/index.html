How it works

One bridge, two machines, no Windows-side AI setup.

01

Start on Mac

Run the local server. It prints LAN pairing URLs and keeps all runtime state on your Mac.

02

Pair Windows

Open the URL on Windows and run the generated PowerShell agent command. The agent polls the Mac.

03

Let AI operate

Use the web console, CLI, and future MCP tools to run jobs, collect logs, inspect screenshots, and install builds.

Built for real Windows work.

The first release focuses on the operations AI agents need when macOS is the thinking environment and Windows is the execution target.

  • PowerShell command queueing
  • Progress logs and exit codes
  • Screenshot capture
  • Artifact serving and installer execution
  • Optional OpenSSH bootstrap
  • CLI-first automation surface

Security model

Local-first by default, token mode when you need it.

WinBridge AI is designed for trusted local networks first. The default pairing flow avoids manual token entry on Windows, while optional token mode can protect the console, API, and agent callbacks when you need a stricter LAN session. 

WINBRIDGE_PORT=47832
WINBRIDGE_STATE_DIR=.state
WINBRIDGE_AUTH_REQUIRED=1
